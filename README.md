# 4、ESP32 蓝牙控制LED跑马灯

## 项目功能
通过手机蓝牙连接ESP32，远程控制多个LED实现跑马灯效果，支持跑马灯开关、速度调节、多种显示模式切换和实时状态反馈。

## 硬件要求
- ESP32开发板
- LED灯珠8个（任意颜色）
- 220Ω限流电阻8个
- 杜邦线若干
- 带蓝牙功能的智能手机
- 手机蓝牙串口APP（推荐：Serial Bluetooth Terminal、蓝牙串口助手、spp蓝牙串口助手）

## 接线说明
| LED序号 | LED阳极(长脚) | ESP32引脚 | LED阴极(短脚) |
|---------|---------------|-----------|---------------|
| 1       | 串联220Ω电阻  | GPIO 2    | GND           |
| 2       | 串联220Ω电阻  | GPIO 4    | GND           |
| 3       | 串联220Ω电阻  | GPIO 16   | GND           |
| 4       | 串联220Ω电阻  | GPIO 17   | GND           |
| 5       | 串联220Ω电阻  | GPIO 5    | GND           |
| 6       | 串联220Ω电阻  | GPIO 18   | GND           |
| 7       | 串联220Ω电阻  | GPIO 19   | GND           |
| 8       | 串联220Ω电阻  | GPIO 23   | GND           |

**重要提示**：每个LED必须单独串联一个220Ω限流电阻，否则会烧毁LED或ESP32引脚。所有LED的阴极需共同连接到ESP32的GND引脚。

## 使用方法
1. 按照接线说明正确连接所有LED和电阻到ESP32开发板
2. 将跑马灯控制代码上传到ESP32开发板
3. 打开手机蓝牙功能，搜索并连接名为`ESP32_BT_MARQUEE`的设备（默认密码：1234）
4. 使用蓝牙串口助手发送以下命令控制跑马灯：
   - 发送 `on`：启动跑马灯效果
   - 发送 `off`：关闭所有LED
   - 发送 `speed 100`：设置跑马灯速度（数值越小速度越快，范围50-1000）
   - 发送 `mode 1`：切换跑马灯模式（支持1-3种不同效果）
   - 发送 `status`：查询当前跑马灯状态和参数

## 关键技术点
- ESP32蓝牙串口通信与命令解析
- 多个GPIO引脚的批量配置与控制
- 非阻塞延时实现流畅的跑马灯动画
- 多模式跑马灯效果的逻辑设计
- 运行参数（速度、模式）的动态修改与保存


# 5、ESP32 SSE+HTTP POST 远程控制LED

## 项目功能
- 局域网内通过任意浏览器网页远程控制ESP32板载LED灯的开关
- 支持多设备同时连接，所有客户端状态实时自动同步
- 浏览器端断线自动重连，无需手动刷新页面
- 响应式控制界面，适配电脑、手机、平板等不同设备
- 纯官方库实现，无需安装任何第三方依赖

## 硬件要求
| 硬件名称 | 数量 | 备注 |
|---------|------|------|
| ESP32开发板 | 1块 | 推荐ESP32-WROOM-32系列 |
| USB Type-C/Micro数据线 | 1根 | **必须是数据传输线**，不能是仅充电线 |
| 电脑 | 1台 | Windows/macOS/Linux均可 |

**额外说明**：本项目默认使用ESP32板载LED，无需额外电子元件。

## 接线说明
### 板载LED（推荐）
绝大多数ESP32开发板的板载LED已默认连接到**GPIO2(D2)**引脚，无需任何额外接线。

### 外接LED（如无板载LED）
- LED正极 → 220Ω限流电阻 → ESP32 D2引脚
- LED负极 → ESP32 GND引脚

## 使用方法
1. **搭建开发环境**
   - 下载并安装Arduino IDE（推荐2.x版本）
   - 打开Arduino IDE → 文件 → 首选项，在"附加开发板管理器网址"中添加：`https://dl.espressif.com/dl/package_esp32_index.json`
   - 工具 → 开发板 → 开发板管理器，搜索并安装"ESP32"开发板支持包（作者：Espressif Systems）
   - 安装对应开发板的串口驱动（CH340或CP2102）

2. **修改代码配置**
   - 复制完整的SSE控制LED代码
   - 将代码开头的`ssid`和`password`替换为你的2.4GHz WiFi名称和密码
   - （可选）修改`ledPin`定义为你的开发板实际LED连接的引脚

3. **上传代码到ESP32**
   - 用USB数据线连接ESP32开发板到电脑
   - 工具 → 开发板，选择对应的ESP32开发板型号（如"ESP32 Dev Module"）
   - 工具 → 端口，选择ESP32对应的COM口
   - 点击右上角的"上传"按钮，等待代码编译并上传完成

4. **查看ESP32 IP地址**
   - 打开Arduino IDE串口监视器
   - 将波特率设置为**115200**
   - 按下ESP32开发板上的复位按钮
   - 等待串口输出"WiFi connected!"和IP地址（如`192.168.1.100`）

5. **局域网内测试控制**
   - 确保你的电脑/手机与ESP32连接到**同一个2.4GHz WiFi网络**
   - 打开浏览器，在地址栏输入ESP32的IP地址
   - 你将看到LED控制界面，点击"开灯"和"关灯"按钮即可控制LED

6. **多客户端同步测试**
   - 在多个设备（电脑、手机、平板）上同时打开控制页面
   - 在任意一个设备上点击按钮，所有设备的界面状态会同步更新
   - LED灯的实际状态与所有界面显示保持一致

## 关键技术点
- **SSE(Server-Sent Events)服务器推送技术**：基于HTTP协议实现服务器主动向客户端推送状态更新，相比HTTP轮询延迟更低、网络开销更小
- **HTTP POST控制指令传输**：客户端通过POST请求向服务器发送控制指令，符合HTTP协议规范
- **纯官方库实现**：仅使用ESP32内置的WiFi库，无需安装任何第三方库，兼容性最好
- **多客户端状态同步**：服务器维护所有SSE连接，状态变化时向所有客户端广播
- **正确的HTTP请求解析**：按行读取HTTP请求头，保留POST请求体，解决了控制指令丢失的问题
- **GPIO电平控制**：通过配置GPIO引脚为输出模式，控制高低电平实现LED开关

# 8、ESP32 UDP板间通信

## 项目功能
实现两块ESP32开发板之间基于UDP协议的无线消息收发功能。

## 硬件要求
- 2块ESP32开发板
- 2根USB数据线

## 接线说明
无需额外接线，使用ESP32板载WiFi功能。

## 使用方法
1. 将`UDP-Sender`代码上传到第一块ESP32（发送端）
2. 将`UDP-Receiver`代码上传到第二块ESP32（接收端）
3. 确保两块ESP32连接到同一个WiFi网络
4. 打开两个串口监视器，波特率设置为115200
5. 发送端会每隔1秒向接收端发送一条消息，接收端收到后会在串口打印出来

## 关键技术点
- UDP协议的基本原理与应用。
- ESP32 WiFi STA模式配置。
- 套接字(Socket)编程基础。
- 网络数据传输与解析。
